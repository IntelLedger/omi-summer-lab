Getting Started¶

This guide will help you get started with Hyperledger Sawtooth and the OMI Summer Lab Transaction Processor.

Getting the Source¶

OMI Summer Lab is made available via the source repository. This repository is hosted on Github.

You will need to have git installed on your system. If you need help setting this up, you’ll find a handy guide here.

Note

On Mac OS, git is already installed, but if you haven’t used it, you may need to run, in the terminal:

$ xcode-select --install

To clone the repository:

$ git clone https://github.com/IntelLedger/omi-summer-lab

Using Docker Compose¶

Docker is an open-source project for running applications in a software container. We provide several docker containers for running the various components of Hyperledger sawtooth, as well as the OMI Summer Lab Transaction Processor. All of these containers can be started in concert by using Docker Compose.

Required Tools¶

The following tools are required:

  • Docker Engine (17.03.0-ce or later)
  • Docker Compose (Linux only)

Starting Up Hyperledger Sawtooth¶

To start a validator node running the OMI transaction handler, run:

% docker-compose -f omi-summer-lab/docker/compose-with-omi.yaml up

This will start a validator, a config transaction handler, an OMI transaction handler, and a REST API instance with which a client can communicate. To stop the validator node, press Ctrl-c and then run:

% docker-compose -f omi-summer-lab/docker/compose-with-omi.yaml down
